# Case Study: Label Printer IP Mismatch and Control Lock

## Environment
- Corperate Warehouse
- Networked label printer
- Site network recently updated with new IP scheme
- Device controls locked to prevent accidental changes
- Printer non-functional for several months

## Initial Observation
- User ticket indicated printer had been offline for months.
- Attempts to source replacement devices had been ongoing.
- Existing documentation mentioned IP address settings and unlock procedure.

## Troubleshooting Steps
1. Verified network settings and IP addressing scheme.
2. Consulted official device documentation for control unlock commands.
3. Manually applied unlock command and set correct IP address on device.
4. Confirmed printer functionality on the network.
